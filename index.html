SQL Injection (SQLi)

Critical

Unsanitized user input in database queries allows attackers to extract or modify data.

Impact: Full database breach, data theft, ransomware, compliance violations (GDPR/PCI fines)
# Use parameterized queries cursor.execute("SELECT * FROM users WHERE id = %s", (user_id,))
// Use prepared statements db.query('SELECT * FROM users WHERE id = ?', [userId])
// Use PDO prepared statements $stmt = $pdo->prepare('SELECT * FROM users WHERE id = ?'); $stmt->execute([$userId]);

Cross-Site Scripting (XSS)

Critical

Malicious scripts injected into web pages execute in users' browsers, stealing sessions or data.

Impact: Session hijacking, credential theft, malware distribution, defacement
// Encode output before rendering element.textContent = userInput; // Safe // NOT: element.innerHTML = userInput; // Unsafe
// React auto-escapes by default <div>{userInput}</div> // Safe // Avoid: dangerouslySetInnerHTML
Content-Security-Policy: default-src 'self'; X-XSS-Protection: 1; mode=block

Broken Authentication

High

Weak session management or credential handling allows account takeover.

Impact: Account takeover, identity theft, unauthorized access to sensitive data
# Regenerate session after login session.regenerate() session['user_id'] = user.id
Set-Cookie: session=abc123; HttpOnly; Secure; SameSite=Strict

Insecure Direct Object Reference

High

Users can access unauthorized resources by manipulating object references (IDs, filenames).

Impact: Unauthorized data access, PII exposure, horizontal privilege escalation
# Always verify ownership def get_document(doc_id, user): doc = Document.query.get(doc_id) if doc.owner_id != user.id: abort(403) # Access denied return doc

Security Misconfiguration

Medium

Default credentials, unnecessary services, or verbose errors expose attack surface.

Impact: System compromise, information disclosure, lateral movement
# Disable debug mode in production DEBUG = False ALLOWED_HOSTS = ['yourdomain.com'] # Hide server version server_tokens off; # nginx

Missing Security Headers

Medium

Absent HTTP headers leave browsers vulnerable to clickjacking, XSS, and MIME attacks.

Impact: Clickjacking attacks, content injection, data exfiltration
# Add to nginx/Apache config X-Frame-Options: DENY X-Content-Type-Options: nosniff Strict-Transport-Security: max-age=31536000 Content-Security-Policy: default-src 'self'

Actively Exploited Vulnerabilities

High-priority CVEs from CISA's Known Exploited Vulnerabilities catalog. Patch these immediately.

Apache Log4j RCE (Log4Shell)

CVE-2021-44228 | CVSS 10.0

Remote code execution via JNDI injection in Log4j logging library. Actively exploited in ransomware attacks.

Fix: Upgrade to Log4j 2.17.1+ or remove JndiLookup class

MOVEit Transfer SQLi

CVE-2023-34362 | CVSS 9.8

SQL injection allowing unauthenticated access. Used by Cl0p ransomware gang in mass exploitation.

Fix: Apply Progress MOVEit patches immediately

Citrix NetScaler RCE

CVE-2023-4966 | CVSS 9.4

Buffer overflow enabling session hijacking. Dubbed "CitrixBleed" - actively exploited in the wild.

Fix: Update to latest NetScaler version, kill active sessions

Microsoft Exchange ProxyShell

CVE-2021-34473 | CVSS 9.8

Pre-auth RCE chain targeting Exchange servers. Widely exploited for webshell deployment.

Fix: Apply KB5001779 cumulative update
